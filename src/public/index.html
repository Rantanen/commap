Connecting to multiple Mumble-servers

Some outfits and players already use Mumble for communication. The following instructions are aimed at these players so they can run Mumble connected to different servers at the same time. These instructions accomplish the following.
  • Connect to two Mumble servers at the same time.
  • Use different push-to-talk keys for the servers.
  • Try to make it as easy as possible.

1. Create a new Mumble copy

Create a copy of the current Mumble folder. This should be somewhere under C:\Program Files\ by default. The new copy can be made anywhere on the computer. In the image below it has been made next to the original Mumble installation.

2. Create a shortcut

Create a shortcut of the new Mumble executable and give it the -m mumble://name@cobalt.jubjubnest.net command line option. The -m lets us launch Mumble a second time and the URL makes it auto-connect to the Cobalt server.

3. Create an empty mumble.ini

Last step to make this new Mumble installation independant of the original one is to create an empty mumble.ini file in the new Mumble folder. This file prevents Mumble from using the settings of the original one from the registry.

Done! -- Well... almost

The new Mumble installation can now be launched from the shortcut and it should open up in its own window. This installation knows nothing of the original one so you will need to configure it just like you did the first time you launched the original Mumble. Below are some things that might be meaningful in some ways.
  • Set up a push-to-talk key.
  • Set up a DIFFERENT push-to-talk key than your other Mumble.
  • Disable the overlay. Does it even work in Planetside? In any case two Mumble versions trying to use the overlay won't end well.
